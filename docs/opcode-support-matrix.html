Collisions & discrepancies

The same byte forks the protocol 0x0082

0x0082 is an NFC endpoint on OD App, Firmware_NRF54 and Firmware_Silabs — but a PIPE_WRITE_END (image-transfer terminator + refresh) on py‑opendisplay, home_assistant and the combined Firmware. Firmware_NRF assigns it nothing. So one wire byte drives an NFC write on Silabs/nRF54 panels but ends an image transfer on ESP32/nRF52 panels — a genuine, load-bearing conflict, not just a naming clash.

PIPE_WRITE is combined-Firmware + py only 0x0080–0x0082

Only py‑opendisplay speaks the sliding-window START/DATA/END, and only the combined Firmware handles all three. Silabs, nRF54 and dedicated nRF have no pipe protocol — a host that streams via pipe to those targets gets “unknown command.” py‑opendisplay gates the attempt on a config advertising bit and falls back to legacy 0x0070 direct-write.

Silently dropped on lean targets 0x0076 · 0x0077

PARTIAL_WRITE_START and BUZZER are sent by OD App + py/HA and handled by combined Firmware and nRF54, but not by dedicated nRF or Silabs — partial-region refresh and buzzer commands vanish there with no error.

Host / firmware mismatches 0x0044 · 0x0045

READ_MSD (0x0044) is issued by OD App and handled by three firmwares, but is defined-but-unhandled on dedicated nRF; py/HA read MSD from the advertisement instead of commanding it. CONFIG_CLEAR (0x0045) is a firmware-only extension on combined Firmware and nRF54 that no host ever sends.

Response codes overlap command space

0x0073/0x0074 also serve as device→host refresh-complete / timeout responses. 0x0074 is never a host→device command in any repo, so it isn’t a matrix row — but the reuse is worth noting when reading raw traces.

Per-repo dispatch

OD Apphost · iOS
Canonical enum OD.Cmd: UInt16 (BLE/ODConstants.swift:23). The bundled JS engine Resources/ble-common.js independently hardcodes the same opcodes as hex strings — the numbers are declared twice. No PIPE_WRITE.
py-opendisplayhost · python
Canonical class CommandCode(IntEnum) (protocol/commands.py:13) with per-opcode build_*_command helpers. The only host that speaks the PIPE_WRITE sliding-window protocol.
home_assistanthost · integration
Defines no opcodes of its own — drives every BLE command through py‑opendisplay’s OpenDisplayDevice. Its opcode support is exactly py‑opendisplay’s.
Firmwarefw · nRF52840 + ESP32
No symbolic defines anywhere — raw 16-bit hex literals inside one dispatcher imageDataWritten() → switch at communication.cpp:583. Widest set; adds 0x0045 and the full PIPE_WRITE trio. 0x0082 = PIPE_WRITE_END, zero NFC code.
Firmware_NRFfw · dedicated nRF
#define CMD_* in EPD/EPD_service.h:67; single switch at EPD_service.c:618. Narrowest firmware — 13 defined, 12 handled. No LED_STOP, partial, buzzer, NFC or pipe.
Firmware_NRF54fw · nordic ncs
Opcodes in opendisplay_protocol.h:6; dispatch() at opendisplay_pipe.c:1178. Handles NFC (0x0082), LED, buzzer, partial and 0x0045; no PIPE_WRITE.
Firmware_Silabsfw · silabs bg22
CMD_* in opendisplay_protocol.h:6; switch at opendisplay_pipe.c:1011. Config, firmware, MSD, auth, reboot, DFU-to-Apploader, EM4 deep-sleep, direct-write trio, LED activate/stop, and NFC. No partial, buzzer or pipe.
